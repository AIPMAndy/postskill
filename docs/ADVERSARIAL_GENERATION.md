# 对抗式内容生成（Adversarial Content Generation）

## 🎯 什么是对抗式生成

对抗式生成是一种通过**三个 AI Agent 协作**来提升内容质量的方法：

```
生成者 → 批判者 → 评估者 → 迭代优化 → 最终高质量内容
  ↑                                              ↓
  └──────────────── 反馈循环 ────────────────────┘
```

### 三个 Agent 的角色

1. **🤖 生成者（Generator）**
   - 职责：根据主题和风格创作内容
   - 特点：创造性强，追求表达
   - 输出：初稿文案 + 创作思路

2. **🔍 批判者（Critic）**
   - 职责：找出内容的所有问题和不足
   - 特点：严格、挑剔、不留情面
   - 输出：问题列表 + 改进建议

3. **⚖️ 评估者（Evaluator）**
   - 职责：客观打分，综合评估质量
   - 特点：公正、理性、标准统一
   - 输出：0-1 分数 + 评分理由

## 🔄 工作流程

### 第 1 轮

```
生成者: 创作初稿
  ↓
批判者: "内容太短，缺少数据支撑，开头不够吸引人"
  ↓
评估者: 0.65 分 - "有明显改进空间"
```

### 第 2 轮

```
生成者: 根据反馈改进（增加数据，优化开头）
  ↓
批判者: "数据不够具体，结尾缺少行动号召"
  ↓
评估者: 0.78 分 - "质量提升，但仍有小瑕疵"
```

### 第 3 轮

```
生成者: 继续优化（具体数据，强化结尾）
  ↓
批判者: "整体优秀，仅有细节可优化"
  ↓
评估者: 0.88 分 - "达到优秀水平" ✅
```

## 🚀 使用方法

### 基础用法

```bash
# 标准生成（快速）
python postskill.py run --topic "AI 创作"

# 对抗式生成（高质量）
python postskill.py run --topic "AI 创作" --adversarial
```

### 高级配置

```bash
# 自定义迭代次数（默认 3 次）
python postskill.py run --topic "AI 创作" --adversarial --iterations 5

# 批量生成 10 套，使用对抗式
python postskill.py run --topic "AI 创作" -c 10 --adversarial

# 使用 GPT-4 + 对抗式（最高质量）
python postskill.py run --topic "AI 创作" -m gpt-4o --adversarial
```

## 💡 最佳实践

### 何时使用对抗式生成

✅ **推荐使用：**
- 重要的营销文案
- 品牌对外传播内容
- 需要高质量的专业文章
- 付费内容、课程文案
- 首页/落地页文案

❌ **不推荐使用：**
- 快速测试、草稿
- 批量生成（>20 套）
- 时间紧急的场景
- 内部沟通文档

### 参数建议

| 场景 | 迭代次数 | 预期质量 | 耗时 |
|------|---------|---------|------|
| 快速优化 | 2 | 0.75+ | 10s |
| 标准质量 | 3 | 0.85+ | 20s |
| 极致质量 | 5 | 0.90+ | 40s |

## 🔧 技术原理

### 为什么对抗式生成质量更高？

1. **多视角审视**
   - 生成者：创造性视角
   - 批判者：挑剔视角
   - 评估者：客观视角

2. **迭代优化**
   - 每轮都基于上一轮反馈改进
   - 问题逐步减少
   - 质量持续提升

3. **标准统一**
   - 评估者使用统一标准打分
   - 避免主观偏差
   - 质量可量化

### 与传统方法对比

| 方法 | 质量 | 速度 | 成本 |
|------|------|------|------|
| 单次生成 | 低 | 快 | 低 |
| 多次重试 | 中 | 慢 | 高 |
| 对抗式生成 | 高 | 中 | 中 |
